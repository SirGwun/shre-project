В Git у файлов может быть 4 статуса 
1) untracked -  неотслеживаемый файл. У таких файлов нет истории изменений, гит видит их, но не следит за ним
2) tracked - Все файлы, в которых гит отслеживает изменнеия, как добавленные add так и закомиченые commit
3) staged - подготовленный, этот файл после выполнения команды git add попадает в список файлов, которые потом попадут в коммит. 
4) modified - означает, что существует предидущая сохраненная версия файла, и файл изменился с тех пор 

Важно оформлять коментарии к комитам понятно и однообразно:
Есть разные стандарты оформления коментариев к комитам приятно - 
1) указывать номер задачи, если используется какой-то трекер задачь
2) использовать инфинитив (исправлено, сделано)
3) указывать в начале тип изменения (fich, fix и тд)
4) По возможности не писать комментарии длиннее 72 символов (стандарно отображаются в log --oneline)
