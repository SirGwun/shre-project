git - любая гит команда начинается с этого слова 
add - добавляет данный файл(ы) к текущему комиту
restor - откатывает файл до опследненго сохраненного стосяния, также используется с --staged чтобы переместить файл из статуса staged обратно в untracked (git restore --staged <file>)
commit - вносит выбраные и добавленые файлы в историю git'a, использовать с -m "сообщение для описанию коммита"
c фагом --ammend можно менять или дополнять последний коммит. При этом можно использовать "git commit --amend --no-edit" чтобы доавить новый файл(ы) в комит и не меня сообщение, или "git commit --amend -m "новое сообщение" чтобы изменить сообщение тоже
reset - команда для откатывания к предидущим комитам (git reset --hard <commit hash>)
push - отправить текущее состояние репозиторию в удаленное хранилище (-u [orirgin] [название ветки] нужно при первом пуше в новый удаленный репозиторий)
status - статус, выводит информацию о состоянии данного гит репозитория
init - создать гит репозиторию в данной директории
remote add [origin] [адрес удаленного репозитория] - отправляет твой локальный репозиторий в удаленный
branch - служит для перемещения по веткам
log - история всех комитов и прочих дествий с некоторой уточнаяющий информацией (автор, датта, сообщение комита). Можно указать флаг --oneline, тогда лог будет выведен в сокращенном формате. 
У каждого комита есть идентефикатор - его hesh. в сокращенном формате хеш сворачивается до минимально необходимого. 
Хеш последнего комита помещается в файл ссылка на который находится в специальном служебном файле git'a - HEAD. HEAD можено использовать в командах git как указатель на последний комит.
